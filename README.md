# L-MAC

L-MAC is an LLM-based feature attribution generation framework.
It includes implementations of several feature selection methods (such as Lasso, Integrated Gradients (IG), and mRMR).

L-MAC can be used for:

* Feature attribution and aggregation

* Attribution-guided prediction

* Model evaluation

This repository provides a fully modular implementation, including core modules such as mapping/, merging/, rule_generation/, and evaluation/, as well as several traditional baseline methods in baselines/.
  

## Project Structure

```bash
L-MAP-main/
├── main.py                     # Main entry point
├── baselines/                  # Baseline methods (IG, Lasso, mRMR, etc.)
├── llm/                        # LLM interface, I/O constraints, prompt construction
├── modules/
│   ├── mapping.py              # Feature mapping
│   ├── merging.py              # Feature merging
│   ├── merging_by_region.py    # Region-based feature merging
│   ├── rule_generation.py      # Attribution rule generation
│   ├── inference.py            # Inference module
│   ├── evaluation.py           # Performance evaluation
│   └── summary.py              # Result summarization
├── utils/                      # Utility functions
```

## Usage

he entry point of the project is main.py.
You can specify the configuration or dataset through command-line arguments.

```bash
python main.py --model 'gpt4o' --setting 'summary' --data_date_version '20250401-30' --pred_num 10
```

You need to configure your LLM API Key in llm/llm_client.py or config/config.py:

```python
OPENAI_API_KEY = "your-api-key"
```

### Outputs

L-MAC generates the following outputs:

* Creative-level attribution generated by the LLM

* Aggregated global attribution

* Performance prediction for new creatives guided by attribution

* Model performance comparison
